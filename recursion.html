RECURSION

Recursion is a programing term that means calling a function from itself. Recursion is best applied when you need to call the same function repeatedly with different parameters from within a loop. While it can be used in many situations, it is most effective for solving problems involving iterative branching, such as fractal math, sorting, or traversing the nodes of complex or non-linear data structures.
One reason that recursion is favored in functional programming languages is that it allows for the construction of code that doesn’t require setting and maintaining state with local variables. Recursive functions are also naturally easy to test because they are easy to write in a pure manner, with a specific and consistent return value for any given input, and no side effects on external variable states. The main gist with recursion though is that with recursion,
  • a function calls itself each time with a smaller piece of input until it reaches something that is called a base case.
  • The base case is the condition when the recursion ends.

The two basic concepts in recursion are:

  1. Smaller input(something that gets us closer to the base case)
  2. The base case (The point where the recursion ends to avoid something like an infinite loop called ‘stack overflow’)

Example 1

You want to write a recursive function that counts down a number from 5 to 1, the function will be written in such a way that the function will print five, then deduct 1 from 5, call itself again and this time print 4. Then it will deduct 1 from 4 again and call itself again to print 3. This will continue to happen until the function gets to 1 then it will stop printing. In this case, 1 is the base case (otherwise it will run into negative numbers and calls itself infinitely until the browser freezes). The syntax will look like this
function countDown (num){
if(num <= 0){
console.log(“all done!”);
return;
}else{
console.log(num);
//reducing the number by one
num - -;
//function recalling itself
countDown(num);
}};
countDown(4);
The base case here is 0.
So if the number becomes less than or equal to zero,
it will print all done in the console,
and the function will stop calling itself.
It is very important to include the return statement. Otherwise the function will continue calling itself. Except this time, rather than print the number, it will continue printing all done until the browser gets frozen.

Example 2

function sumRange(num){
if (num === 1) return 1;
return num + sumRange(num – 1);
}
sumRange(3);
The function will return 6
The function took 3
added it to 3-1
and added it to 2-1
Which implies 3+2+1
The base case is 1 because when num becomes equal to one
the function returns 1 and stops running.

Summary

  • A recursive function is a function that calls itself
  • A recursive function calls itself with a different input each time, which is usually a smaller piece of the previous input
  • A recursive should always have a base case, that is the point where the function returns and stops recalling itself
  • If there is no base case, the function will continue to call itself until the browser freezes, this is call stack overflow.
